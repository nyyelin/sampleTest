အလုပ်မလျောက်ခင်၊ အင်တာဗျူးမသွားခင် ပြင်ဆင်ထားရမည့် အချက်များ


UI/UX ကို လေ့လာပြီးအလုပ်လျောက်တော့မယ်လို့ စဉ်စားထားပြီဆိုတာနဲ့ ကျန်တော်တို့ မဖြစ်မနေ အလုပ်လျောက်ဖို့အတွက်  ကြိုတင် ပြင်ဆင်ရတော့မှာပါ။အလုပ်လျောက်ဖို့နဲ့အင်တာဗျူးအတွက် ဘာတွေကြိုတင်ပြင်ဆင်ရမှန်း မသိလို စဉ်းစားရခက်နေပြီဆိုရင် ခု ရေးသားထားတဲ့ အကြောင်းအရာလေးတွေက စာဖတ်သူအတွက် အကူအညီဖြစ်မယ်လို့ထင်ပါတယ်။
ကျန်တော် Junior အနေနဲ့အလုပ်လျောက်တုန်းကပြင်ဆင်ခဲ့ရတဲ့ အတွေ့ကြုံလာခဲ့အပေါ်အခြေခံပြီး ပြင်ဆင်သွားရမယ့်အချက်များကို ဖော်ပြပေးပါမယ်။ပြင်ဆင်ရမယ့် အချက်လေးတွေကို  မပြောခင် ကျန်တော်တို့ အရင်ဆုံး  အကြံပေးချင်တာက အလုပ်ခေါ်စာတွေကို အရင်လိုက်ဖတ်ပါ။အလုပ်ခေါ်စာတွေမှာ ကိုယ်မသိတဲံ JD/JR တွေ ပါကောင်းပါနိုင်ပါတယ်။ မသိတာတွေ နားမလည်တာတွေပါလာခဲ့ရင် အရင်ဆုံး သိအောင်လုပ်ဖို့အကြံပေးချင်ပါတယ်။ သိအောင်လုပ်ခြင်းအားဖြင့် ကိုယ့် skills ကိုလည်း ပိုကောင်းစေပါတယ်။

(1) Portfolio


ပထမအရေးကြီးဆုံးကတော့ Portfolio ပဲဖြစ်ပါတယ်။ အလုပ်လျောက်တဲ့အခါ တော်တော်များများက Portfolio  တောင်းတတ်ကြပါတယ်။ ဒါကြောင့် Portfolio ကို UI/UX Creator တစ်ယောက်အနေဲ့ ဖန်တီးမှုးကောင်းကောင်းနဲံ  ပြင်ဆင်ထားဖို့လိုပါတယ်။Portfolio အတွက်ကတော့ ကျောင်းမှာလုပ်ထားတဲ့ projects တွေ လည်းပြလို့ရပါတယ်။ကိုယ်က  ကျောင်းမှာ လုပ်ထားတာမရှိဖူးဆိုရင်လည်း သင်တန်းမှာလုပ်ထားတဲ့ Projects တွေကိုလည်း ပြလို့ရပါတယ်။သတိထားရမှာက ကိုယ့် Projects တွေကို သေသေချာချာ သပ်သပ်ရပ်ရပ် ပြင်ဆင်ထားဖို့လိုအပ်ပါတယ်။Portfolio မလုပ်တတ်ဘူးဆိုရင်တော့  Pinterest / Behance တို့အပြင် Free Portfolio Template တွေမှာ သူများလုပ်ပြီးသား Design တွေကို လေ့လာပြီး Reference ယူလို့ ရပါတယ်။ကိုယ့်ဘာသာလုပ်လို့ အဆင်မပြေရင်တော့လို့ အကြံဉာဏ်လိုချင်ရင် Seniors တွေ ၊ သက်ဆိုင်ရာ နည်းပညာGroups တွေမှာ မေးလို့ရပါတယ်။

(2) CV / Resume / Cover Letter


နောက်တစ်ခုအနေနဲ့ အရေးကြီးတာကတော့ CV / Resume ပဲဖြစ်ပါတယ်။ဒီနေရာမှာ CV နဲ့ Resume ကို ခွဲခြားတတ်ဖို့လိုပါတယ်။ ကိုယ်အလုပ်လျောက်မယ့် Companyက ပေါ်မူတည်ပြီး ပြင်ဆင်ပေးရမှာပါ။Local မှာတော့ CV နဲ့ Resume ကို တွဲသုံးနေတာများပါတယ်။ ကိုယ်က CV / Resume ကို မပြင်တတ်ဖူး၊ မရေးတတ်ဘူး ဆိုရင်လည်း online မှာ sample တွေ အများကြီးရှိပါတယ်။ www.canva.com/resumes/ မှာ လည်း  ကိုယ့်ကြိုက်တဲ့ design ကို ယူပြီးပြင်လို့ရပါတယ်။တစ်ခါတစ်လေ cover letter ကို တာင်းတတ်တာမလို့   ပြင်ဆင်ထားရင် မမှားဘူးလို့ထင်ပါတယ်။ Cover Letter အနေနဲံ စာမျက်နှာ တစ်မျာက်နှာအတွင်း စကားလုံး စာလုံးရေ ၂၅၀ မှ ၄၀၀လောက်ထိဆို အဆင်ပြေပါတယ်။

(3) UX Case Study / Redesign


Website တွေ App တွေ ကိုလည်း Case Study / Redesign တွေလုပ်ဖို့လည်း လိုပါတယ်။ နာမည်ကြီး Webiste တွေ App တွေရဲ့ Process အကုန်လုံးကို မလုပ်နိုင်ရင်တောင်မှ ကိုယ်ကြိုက်တဲံ အပိုင်းကို ရွေးပြီး Case study / redesign တွေလုပ်လို့ရပါတယ်။လုပ်ပြီးရင်လည်း ကျန်တော်တို့ Dribbble / Behance တို့အပြင် UI/UX Group တွေမှာတင်ပြီး Senior တွေ  ရဲ့ Feedback ကိုပြန်ယူလို့ရပါသေးတယ်။ Case Study အပိုင်းနဲံပတ်သတ်ပြီးလေ့လာချင်ရင်တော့ Lollypop / Iman / Tanyaoxley တို့မှာ သွားရောက်လေ့လာလို့ရပါတယ်။

(4)Coding skill


UI / UX သမားတစ်ယောက်အနေနဲ့ Code (Html,Css,Bootstrap,MDB...)ရေးစရာမလိုဘူးဆိုပေမယ့် သိထားရင် ရေးတတ်ရင် ပိုအဆင်ပြေပါတယ်။ Frontend developer တွေနဲ့   ပူပေါင်းလုပ်တဲ့အခါမှာ တော်တော်အဆင်ပြေတာကို လုပ်ငန်းခွင် ရောက်တဲ့အခါတွေ့ရမှာပါ။ပြောရရင် UI/UX designer နဲ့ Frontend developer က သမီးရည်စားနဲ့တူပါတယ် (ရန်ဖစ်လိုက် ပြန်ချစ်လိုက်နဲ မတည့်အတူနေရတာပါ)။ ခုနောက်ပိုင်း Local အတွင်း အလုပ်ခေါ်စာတွေမှာ  အတွေ့များတာကတော့ UI/UX လို့သာအမည်ခံပြီး Graphic Design / UI/UX / Front end ပါစွယ်စုံရတာမျိူးကို လိုချင်ကြတာပါ။အဲတာကြောင့်သိထားတာအကောင်းဆုံးလိုထင်ပါတယ်။

(5) Company အကြောင်း


Interview မသွားခင်မှာ ကိုယ်သွားဖြေမယ့် company ရဲ့ အကြောင်းကို သိထားဖို့လိုအပ်ပါတယ်။ company အကြောင်းကို ကျန်တော်တို့ google ခေါက်ပြီးရှာဖွေလို့ ရသလို၊ Facebook မှာလည်း သွားရောက်ကြည့်လို့ရပါတယ်။ကိုယ်အလုပ်လျောက်တဲ့ Company က ဘယ်productsတွေ ထုတ်တာလဲ၊ ဘယ်လုပ်ငန်းတွေကို အဓီက လုပ်ကိုင်နေတာလဲဆိုတာ  သိထားမှ  company အကြောင်းသိလား မေးလာမယ်ဆိုရင် ကိုယ်က  ဖြေဆိုနိုင်မှာဖြစ်ပါတယ်။သူတို့ company ကို ဘယ်လောက်  စိတ်ဝင်စားမှုရှိလဲဆိုပြီး မေးမြန်းတဲ့သဘောဖြစ်ပါတယ်။လေ့လာထားတဲ့သူဆိုရင် company ဘက်ကလည်းသဘောကျပါတယ်။ ( Company က Family Type ဆိုရင်တော့ သတိတော့ထားနော်!)

(5)  Interview Test


ကျန်တော်တို့ အလုပ်လျောက်ပြီးတော့  interview ဖြေဆိုမယ်ဆိုရင် skill test က ပါကောင်းပါနိုင်ပါတယ်။ဒါကြောင့် ကျန်တော်တို့ မသွားခင်မှာ ပြင်ဆင်သွားဖို့ လိုအပ်ပါတယ်။UI/UX position အနေနဲ့ပြောရမယ်ဆိုရင် Interview မှာ Design Principles and  Theoryပိုင်းတွေ (eg. Color Theory / UX Theory / UX Process ) ကို မေးတတ်တာမလို့ ပြန်လေ့လာပြီးမှ သွားဖြေတာအကောင်းဆုံးပါပဲ။ Design Test အနေနဲကတော့ Design မှာပါဝင်မည့်အကြောင်းအရာများကို ပေးပြီး ၂ရက် (or) ၃ရက်  အချိန်ပေးပြီးဆွဲခိုင်းလေ့ရှိပါတယ်။

(6) Job Description / Salary


Interview မှာ HR/Team Leader တို့မေးတဲ့အလှည့်ပြီးရင်  ကိုယ့်ကို ဘာမေးစရာရှိသေးလဲလို့ မေးလေ့ရှိပါတယ်။မမေးဘူးဆိုရင်လည်း ကိုယ်နားမလည်တာတွေ မသိတာတွေကို အဲအချိန်မှာ မေးစရာရှိတာတွေကို တစ်ခါတည်းမေးပါ။ ကိုယ့် JD ကို ရှင်းအောင် သေချာမေးပါ။ တစ်ချို့ Company တွေမှာ Family type ဆိုရင်တော့ JD အပြင်မဆိုင်တာတွေပါ  ခိုင်းတတ်ပါတယ် (ကိုယ်တွေ့ပါ)။ နောက် Salary နဲ့ပတ်သတ်ရင်လည်း သေသေချာချာမေးဖို့လိုအပ်ပါတယ်( ၃လပြည့်ရင် လစာတိုးပေးမယ် ဆိုတာမျိူးကိုဆိုလိုတာပါ ) ။အကယ်ရျ် လစာတိုးပေးမယ်လို့ပြောရင်တော့  HR မမတွေ ခဏခဏသတိပေးဖို့မမေ့ပါနဲ့ မေ့သွားခဲ့ရင် အလုပ်ကမထွက်မချင်း လစာမတိုးတတ်တာမျိူးတွေ ရှိတတ်ပါတယ်။
